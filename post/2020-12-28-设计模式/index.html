设计模式

单例模式Singleton

创建型模式。

其特点是只提供唯一一个类的实例,具有全局变量的特点，在任何位置都可以通过接口获取到那个唯一实例。

适用场景：

  1. 设别管理器，系统中可能有多个设备，但是只有一个设备管理器，用于管理设备驱动；
  2. 数据池，用来缓存数据的数据结构，需要在一处写，多处读取或者多处写，多处读取。

特点：

  1. 全局只有一个实例：static特性，同时禁止用户自己声明并定义（把构造函数写为private）
  2. 线程安全；
  3. 禁止赋值和拷贝；
  4. 用户通过接口获取实例：使用static类成员函数

好处：

  1. 保证全局唯一性；
  2. 节约系统资源；
  3. 避免对共享资源的多场占用。

样例：

Meyers' Singleton

#include <iostream>

class Singleton
{
private:
    Singleton() { std::cout << "constructor called!" << std::endl; }

public:
    Singleton(const Singleton &) = delete;
    ~Singleton() { std::cout << "destructor called!" << std::endl; }

public:
    Singleton &operator=(const Singleton &) = delete;
    static Singleton &GetInstance()
    {
        static Singleton instance;
        return instance;
    }
};

int main(int argc, char *argv[])
{
    Singleton &instance_1 = Singleton::GetInstance();
    // Singleton &instance_2 = Singleton::GetInstance();
    return 0;
}

观察者模式Observer/Event

行为型模式。

定义对象间的一种一对多（变化）的依赖关系，以便当一个对象（Subject）的状态发生变化时，所有依赖于它的对象都得到通知并自动更新。

实现软件体系结构的松耦合。

模式的结构

观察者模式的主要角色如下：

  1. 抽象主题（Subject）角色：也叫抽象目标类，它提供了一个用于保存观察者对象的聚集类和增加、删除观察者对象的方法，以及通知所有观察者的抽象方法。
  2. 具体主题（Concrete Subject）角色：也叫具体目标类，它实现抽象目标中的通知方法，当具体主题的内部状态发生改变时，通知所有注册过的观察者对象。
  3. 抽象观察者（Observer）角色：它是一个抽象类或接口，它包含了一个更新自己的抽象方法，当接到具体主题的更改通知时被调用。
  4. 具体观察者（Concrete Observer）角色：实现抽象观察者中定义的抽象方法，以便在得到目标的更改通知时更新自身的状态。

优点：

  1. 降低了目标与观察者之间的耦合关系，两者之间是抽象耦合关系。符合依赖倒置原则。
  2. 目标与观察者之间建立了一套触发机制。

缺点：

  1. 目标与观察者之间的依赖关系并没有完全解除，而且有可能出现循环引用。
  2. 当观察者对象很多时，通知的发布会花费很多时间，影响程序的效率。

模式的实现

#include <iostream>
#include <list>
#include <string>
using namespace std;

// 抽象观察者.
class Observer
{
public:
    // 纯虚函数.
    virtual void Update() = 0;
};

// 主题或抽象通知者，一般用一个抽象类或者接口实现
class Subject
{
private:
    // 维护一个观察者列表.
    list<Observer *> observers;

public:
    // 绑定观察者.
    inline void Attach(Observer *observer) { this->observers.push_back(observer); }
    // 解绑观察者.
    inline void Detach(Observer *observer) { this->observers.remove(observer); }
    // 通知所有的观察者更新.
    inline void Notify()
    {
        for (auto observer : observers)
        {
            observer->Update();
        }
    }
};

class ConcreteSubject : public Subject
{
private:
    // 主题状态.
    string subjectState;

public:
    inline string GetState() { return subjectState; }
    inline void SetState(string state) { this->subjectState = state; }
};

class ConcreteObserver : public Observer
{
private:
    string name;              // 观察者姓名.
    ConcreteSubject *subject; // 观察者观察的主题.
    string observerState;     // 观察者状态.

public:
    // 构造函数.
    ConcreteObserver(string n, ConcreteSubject *s) : name(n), subject(s) {}

public:
    // 更新函数.
    virtual void Update()
    {
        observerState = this->subject->GetState();
        cout << "observer: " << this->name << " updated, its new state is: " << this->observerState << endl;
    }

    inline string GetState() { return this->observerState; }
    inline void SetState(string state) { this->observerState = state; }
};

int main()
{
    ConcreteSubject *s = new ConcreteSubject();
    s->SetState("ABC");

    ConcreteObserver *ox = new ConcreteObserver("x", s);
    ConcreteObserver *oy = new ConcreteObserver("y", s);
    ConcreteObserver *oz = new ConcreteObserver("z", s);

    s->Attach(ox);
    s->Attach(oy);
    s->Attach(oz);
    s->Notify();

    s->SetState("XYZ");
    s->Notify();

    delete ox;
    delete oy;
    delete oz;
    delete s;

    return 0;
}

工厂模式Factory

创建型模式。

工厂方法Factory Method

模式的实现

#include <iostream>
#include <string>
using namespace std;

/**
 * Product 接口提供了所有 ConcreteProduct 都要实现的操作.
 */

class Product
{
public:
    virtual string Operation() const = 0;
    virtual ~Product(){};
};

/**
 * ConcreteProduct 为 Product 接口提供多种实现.
 */
class ConcreteProduct1 : public Product
{
public:
    virtual string Operation() const override
    {
        return "{Result of the ConcreteProduct1}";
    }
};

class ConcreteProduct2 : public Product
{
public:
    string Operation() const override
    {
        return "{Result of the ConcreteProduct2}";
    }
};

/**
 * Creator 类声明了要返回 Product 类对象的工厂方法. Creator 的派生类通常要实现
 * 这个方法.
 */

class Creator
{
    /**
     * 请注意, Creator 类也许会提供一些默认的工厂方法的实现, 例如 SomeOperation().
     */
public:
    virtual Product *FactoryMethod() const = 0;
    /**
     * 也请注意, Creator 的主要任务不是创建 Product. 通常, 其中会包含一些依赖于
     * 由工厂方法返回的 Product 对象的业务逻辑. 派生类可以通过重写工厂方法并从中
     * 返回不同类型的 Product 来间接地更改业务逻辑.
     */

    // 虚基类也可以提供具体函数实现.
    string SomeOperation() const
    {
        // 调用工厂方法创建一个 Product 对象.
        Product *product = this->FactoryMethod();
        // 使用 Product 对象.
        string result = "Creator: The same creator's code has just worked with " + product->Operation();
        delete product;
        return result;
    }

public:
    // 注意基类的析构函数一定是虚函数.
    virtual ~Creator() {}
};

/**
 * 具体的 Creator 重写工厂方法以更改结果中的 Product 类型.
 */
class ConcreteCreator1 : public Creator
{
    /**
     * 注意, 即使从工厂方法返回的对象为 Concrete Product, 但函数的返回值仍是 Product.
     * 这样做能使得 Creator 独立于 Concrete Product 类.
     */
public:
    Product *FactoryMethod() const override
    {
        return new ConcreteProduct1();
    }
};

class ConcreteCreator2 : public Creator
{
public:
    Product *FactoryMethod() const override
    {
        return new ConcreteProduct2();
    }
};

/**
 * 尽管还是通过基类的接口, 但客户代码使用具体的 Creator 实例工作. 只要客户端继续
 * 通过基类接口, 就可以通过其传递任何 Creator 的派生类.
 */
void ClientCode(const Creator &creator)
{
    cout << "Client: I'm not aware of the creator's class, but it still works.\n"
         << creator.SomeOperation() << endl;
}

/**
 * 程序根据配置或环境选择一种创建者类型.
 */

int main()
{
    cout << "App: Launched with the ConcreteCreator1.\n";
    Creator *creator = new ConcreteCreator1();
    ClientCode(*creator);
    cout << endl;

    cout << "App: Launched with the ConcreteCreator2.\n";
    Creator *creator2 = new ConcreteCreator2();
    ClientCode(*creator2);

    delete creator;
    delete creator2;

    return 0;
}
