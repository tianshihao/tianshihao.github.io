STL 面经📚

2021-01-09
面经 > C++ > STL

0. 概述

STL六大部件：容器、算法、迭代器、函数对象（仿函数）、适配器和分配器。

0.1 容器，Container

一些封装数据结构的模板类，例如 vector 向量容器、list 列表容器等。

分为顺序容器和关联容器，顺序容器有

  • vector
  • deque
  • list
  • array（C++11）
  • forward_list（C++11）

关联容器有

  • 基于红黑树
    • set、multiset
    • map、multimap
  • 基于哈希表分离链接法
    • unordered_set、ordered_multiset（C++11）
    • unordered_map、unordered_multimap（C++11）

0.2 算法，Algorithm

STL 提供了非常多（大约 100 个）的数据结构算法，它们都被设计成一个个的模板函数，这些算法在 std 命名空间中定义，其中大部分算法都包含在头文件 <algorithm> 中，少部分位于头文件 <numeric> 中。

0.3 迭代器，Iterator

对容器中数据的读和写，是通过迭代器完成的，扮演着容器和算法之间的胶合剂。

0.4 函数对象，Function Objects

如果一个类将 () 运算符重载为成员函数，这个类就称为函数对象类，这个类的对象就是函数对象，故又称仿函数（Functor）。

0.5 适配器，Adapter

可以使一个类的接口（模板的参数）适配成用户指定的形式，从而让原本不能在一起工作的两个类工作在一起。值得一提的是，容器、迭代器和函数都有适配器。

STL 提供了 3 种容器适配器，分别为 stack 栈适配器、queue 队列适配器以及 priority_queue 优先权队列适配器。

0.6 分配器，Allocator

为容器类模板提供自定义的内存申请和释放功能，由于往往只有高级用户才有改变内存分配策略的需求，因此内存分配器对于一般用户来说，并不常用。

1. 容器，Container

1.1 std::vector，封装的动态数组

1.1.1 概述

定义于头文件 <vector>。

std::vector 是封装动态数组的序列容器。vector 的存储是自动管理的，按需扩张收缩。vector 通常占用多于静态数组的空间，因为要分配更多内存以管理将来的增长，如果没有剩余空间了，则会重新配置原有元素个数的两倍空间，然后将原空间元素通过复制的方式（内存拷贝）初始化新空间，再向新空间增加元素，最后析构并释放原空间，之前的迭代器会失效。在堆上分配内存。

1.1.2 vector 上的常见操作复杂度（效率）

  • 随机访问——常数 O(1)
  • 在末尾插入或移除元素——均摊常数 O(1)
  • 插入或移除元素——与到 vector 结尾的距离成线性 O(n)

1.1.3 使用场景

需要高效的随机存取，且不经常对非尾结点进行插入、擦除或不在乎插入、擦除的效率。

1.1.4 迭代器非法化

操作 非法化
所有只读操作 决不
swap 、 std::swap end()
clear 、 operator= 、 assign 始终
reserve 、 shrink_to_fit 若 vector 更改容量，则为其全部。否则为无。
erase 被擦除元素及之后的所有元素（包括 end() ）。
push_back 、 emplace_back 若 vector 更改容量，则为其全部。否则仅 end() 。
insert, emplace 若 vector 更改容量，则为其全部。否则仅为在或于插入点后者（包括 end() ）。
resize 若 vector 更改容量，则为其全部。否则仅 end() 与被擦除元素。
pop_back 被擦除元素及 end() 。

1.1.5 创建和初始化

// 初始化.
// 声明一个空 vector 容器, 因为容器中没有元素, 所以没有为其分配空间.
std::vector<double> values;
// 在空容器的基础上, 调用 reverse 函数增加容器容量, 后续最好更新迭代器.
values.reserver(20);
// 在创建容器时初始化, 指定初始值和元素个数.
std::vector<int> primes {2, 3, 5, 7, 11, 13, 17, 19};
// 在创建容器时指定元素个数, 此时容器有 20 个元素, 默认初始值都为 0.
std::vector<double> values(20);
// 若不想初始值为 0, 则也可以指定一个值. 20 个元素值都是 1.0.
std::vector<double> values(20, 1.0);

// 通过其它 vector 创建新的 vector.
int array[]={1,2,3};
// values 将保存 {1, 2}.
std::vector<int>values(array, array + 2);
std::vector<int>values1{1, 2, 3, 4, 5};
// values2 保存 {1, 2, 3}.
std::vector<int>values2(std::begin(values1), std::begin(values1) + 3);

1.2 std::deque，两端都可以扩充的双端队列

1.2.1 概述

定义于头文件 <deque>。

std::deque （double-ended queue）是有下标的顺序容器，它允许在其首尾两端快速插入及删除。另外，在 deque 任一端插入或删除不会非法化指向其余元素的指针或引用。

与 std::vector 相反，deque 的元素不是相接存储的：典型实现用单独分配的固定大小数组的序列，外加额外的登记，这表示下标访问必须进行二次指针解引用，与之相比 vector 的下标访问只进行一次。

deque 的存储按需自动扩展及收缩。扩张 deque 比扩张 std::vector 更优，因为它不涉及到复制既存元素到新内存位置。另一方面， deque 典型地拥有较大的最小内存开销；只保有一个元素的 deque 必须分配其整个内部数组（例如 64 位 libstdc++ 上为对象大小 8 倍； 64 位 libc++ 上为对象大小 16 倍或 4096 字节的较大者）。

能用 vector 尽量使用 vector，通过deque用来做队列即可。

deque 是分段连续。段内是连续的。deque 内部有一个中控器，（map并不是STL中的map）是一小块连续的空间，每个元素都是一个指针，维护每段缓冲区（比较大的连续线性空间）中的内容，缓冲区中存放大量真实数据。中控器维护的是每段缓冲区的地址，使得使用 deque 时像一片连续的内存空间。

1.2.2 deque 上常见操作的复杂度（效率）

  • 随机访问——常数 O(1)
  • 在结尾或起始插入或移除元素——常数 O(1)
  • 插入或移除元素——线性 O(n)

1.2.3 使用场景

双端队列，需要随机存取，而且关心两端数据的插入和删除。

1.2.4 迭代器非法化

操作 被非法化
所有只读操作 决不
swap 、 std::swap 尾后迭代器可能被非法化（实现定义）
shrink_to_fit 、 clear 、 insert 、 emplace 、 push_front 、 push_back 、 emplace_front 、 emplace_back 始终
erase 若在起始擦除——仅被擦除元素 若在末尾擦除——仅被擦除元素和尾后迭代器 否则——非法化所有迭代器（包含尾后迭代器）。
resize 若新大小小于旧者：仅被擦除元素和尾后迭代器 若新大小大于旧者：非法化所有迭代器 否则——不非法化任何迭代器。
pop_front 仅有指向被擦除元素者
pop_back 仅有指向被擦除元素者和尾后迭代器
非法化注意
  • 从 deque 任一端插入时， insert 和 emplace 不会非法化引用。
  • push_front 、 push_back 、 emplace_front 和 emplace_back 不会非法化任何到 deque 元素的引用。
  • 从 deque 任一端擦除时， erase 、 pop_front 和 pop_back 不会非法化到未擦除元素的引用。
  • 以较小的大小调用 resize 不会非法化任何到未擦除元素的引用。
  • 以较大的大小调用 resize 不会非法化任何到 deque 元素的引用。

1.2.5 示例

#include <deque>
#include <iostream>

int main()
{
    std::deque<int> d = {7, 5, 16, 8};

    d.push_front(13);
    d.push_back(25);

    for (int n : d)
    {
        std::cout << n << "\n";
    }

    return 0;
}

1.3 std::list，双向链表

1.3.1 概述

定义于头文件 <list>。

std::list，又称双向链表容器，环状，是支持常数时间从容器任何位置插入和移除元素的容器。不支持快速随机访问。它通常实现为双向链表。在堆上分配内存，每插入一个元数都会分配空间（不是内存拷贝），每删除一个元素都会释放空间。

创建同 vector。

1.3.2 list 上的常见操作复杂度（效率）

  • 随机访问——常数 O(n)
  • 在末尾插入或移除元素——均摊常数 O(1)
  • 插入或移除元素——与到 list 结尾的距离成线性 O(1)

1.3.3 使用场景

需要高效的插入和删除，而不关心随机访问。

1.3.4 迭代器非法化

在 list 内或在数个 list 间添加、移除和移动元素不会非法化迭代器或引用。迭代器仅在对应元素被删除时非法化。

1.3.5 示例

#include <algorithm>
#include <iostream>
#include <list>

int main()
{
    // 创建含有整数的 list.
    std::list<int> l = {7, 5, 16, 8};

    // 添加整数到 list 开头.
    l.push_front(25);

    // 添加整数都 list 结尾.
    l.push_back(13);

    // 搜索插入到 16 前的值.
    auto it = std::find(l.begin(), l.end(), 16);
    if (it != l.end())
    {
        // 在迭代器 it 前插入整数.
        l.insert(it, 42);
    }

    // 迭代并打印 list 的值.
    for (int n : l)
    {
        std::cout << n << std::endl;
    }

    return 0;
}

1.4 std::array，普通数组增强版

1.4.1 概述

定于于头文件 <array>。

std::array 是封装固定大小数组的容器，无法动态的扩展或收缩，在使用该容器的过程无法借由增加或移除元素而改变其大小，它只允许访问或者替换存储的元素。在栈上分配内存。介于传统数组和 vector 之间，封装了一些函数，比传统数组方便，但又没必要使用 vector。此容器是一个聚合类型，其语义等同于保有一个 C 风格数组 T[N] 作为其唯一非静态数据成员的结构体，它不会自动退化成指针（或被隐式的转换成指针），定义时需要指定类型和大小。该结构体结合了 C 风格数组的性能、可访问性与容器的优点，比如可获取大小、支持赋值、随机访问迭代器等。

1.4.2 list 上的常见操作复杂度（效率）

  • 随机访问——常数 O(1)
  • 无法增加或移除元素

“由于 array 容器中包含了 at() 这样的成员函数，使得操作元素时比普通数组更安全，且效率没有因此变差。”

1.4.3 使用场景

  • 知道

1.4.4 迭代器非法化

按照规则，指向 array 的迭代器在 array 的生存期间决不非法化。然而要注意，在 swap 时，迭代器将继续指向同一 array 的元素，并将改变元素的值。

1.4.5 创建和初始化

// 创建了一个 array 容器, 包含 10 个浮点型元素, array 不会初始化, 元素值不确定.
std::array<double, 10> values1;
// 初始化为 0 或和默认元素类型等效的值.
std::array<double, 10> values2 {};
// 注意, 只初始化了前 4 个元素, 剩余元素都被初始化为 0.0.
std::array<double, 10> values3 {0.5, 1.0, 1.5, 2.0};
// 可以使用成员函数 fill 对所有元素进行赋值, 会覆盖已有数据.
vaules3.fill(0);

1.4.6 示例

#include <algorithm>
#include <array>
#include <iostream>
#include <iterator>
#include <string>

int main()
{
    // 用聚合初始化构造
    std::array<int, 3> a1{{1, 2, 3}};  // CWG 1270 重申前的 C++11 中要求双花括号
                                       // （ C++11 之后的版本和 C++14 起不要求）
    std::array<int, 3> a2 = {1, 2, 3}; // = 后决不要求
    std::array<std::string, 2> a3 = {std::string("a"), "b"};

    // 支持容器操作
    std::sort(a1.begin(), a1.end());
    std::reverse_copy(a2.begin(), a2.end(),
                      std::ostream_iterator<int>(std::cout, " "));

    std::cout << '\n';

    // 支持带范围 for 循环
    for (const auto &s : a3)
    {
        std::cout << s << ' ';
    }

    return 0;
}

1.5 std::forward_list，单链表

1.5.1 概述

定义于头文件 <forward_list>。

std::forward_list 是支持从容器中的任何位置快速插入和移除元素的容器。不支持快速随机访问，也是标准库容器中唯一一个不提供 size() 方法的容器。。它实现为单链表，且实质上与其在 C 中实现相比无任何开销。与 std::list 相比，此容器在不需要双向迭代时提供更有效地利用空间的存储。

1.5.2 forward_list 上的常见操作复杂度（效率）

  • 随机访问——常数 O(1)
  • 在末尾插入或移除元素——均摊常数 O(1)
  • 插入或移除元素——与到 forward_list 结尾的距离成线性 O(n)

1.5.3 使用场景

存储相同个数的同类型元素，单链表耗用的内存空间更少，空间利用率更高，并且对于实现某些操作单链表的执行效率也更高。

效率高是选用 forward_list 而弃用 list 容器最主要的原因，换句话说，只要是 list 容器和 forward_list 容器都能实现的操作，应优先选择 forward_list 容器。

1.5.4 迭代器非法化

在链表内或跨数个链表添加、移除和移动元素，不会非法化当前指代链表中其他元素的迭代器。然而，在从链表移除元素（通过 erase_after）时，指代对应元素的迭代器或引用会被非法化。

1.6 std::set，集合

1.6.1 概述

包含在头文件 set，类模板定义如下：

template<
	class Key,                                // 键 key 和值 value 的类型
	class Compare =std::less<T>,              // 指定 set 容器内部的排序规则
	class Allocator = std::allocator<Key>     // 指定分配器对象的类型
> class set;

对于 set 类模板中的 3 个参数，后 2 个参数自带默认值，且几乎所有场景中只需使用前 2 个参数，第 3 个参数不会用到。

集合是一个简单直观的数学概念——具有共同特征的事物的集合。

std::set 是关联容器，含有 Key 类型对象的已排序集。用比较函数比较（Compare）进行排序。搜索、移除和插入拥有对数复杂度。 set 通常以红黑树实现。

更重要的是，从语法上讲 set 容器并没有强制对存储元素的类型做 const 修饰，即 set 容器中存储的元素的值是可以修改的。但是，C++ 标准为了防止用户修改容器中元素的值，对所有可能会实现此操作的行为做了限制，使得在正常情况下，用户是无法做到修改 set 容器中元素的值的。

对于初学者来说，切勿尝试直接修改 set 容器中已存储元素的值，这很有可能破坏 set 容器中元素的有序性，最正确的修改 set 容器中元素值的做法是：先删除该元素，然后再添加一个修改后的元素。

1.6.2 set 上的常见操作复杂度（效率）

  • 随机访问——对数 O(\(log_2n\))
  • 插入或移除元素——对数 O(\(log_2n\))

1.6.3 使用场景

快速查找，不允许重复键值。

1.6.4 迭代器非法化

如果迭代器所指向的元素被删除，则该迭代器失效。其它任何增加、删除元素的操作都不会使迭代器失效。

1.6.5 创建和初始化

std::set<std::string> mySet; // 调用默认构造函数, 创建空的 set 容器.

// 在创建的同时初始化. 内部顺序为 "apple", "banana", "peach".
std::set<std::string> fruit {
    "banana",
    "apple",
    "peach"
};
std::set<std::string> copySet(mySet);
std::set<std::string> copySet = mySet;

// 支持取已有 set 容器中的部分元素, 来初始化新 set 容器, 例如一下, set 中包含 "apple" 和 "peach" 2 个 string 字符串.
std::set<std::string> copySet(++mySet.begin(), mySet.end());

// 借助 set 类模板定义中第 2 个参数, 可以手动修改 set 容器中的排序规则. 内部顺序为 "peach", "banana", "apple".
std::set<std::string, std::greater<string>> copySet {
    "banana",
    "apple",
    "peach"
};

1.7 std::multiset，可重复集合

1.7.1 概述

定义头文件 <set>，类模板定义如下：

template<
	class Key,                                // 键 key 和值 value 的类型
	class Compare =std::less<T>,              // 指定 set 容器内部的排序规则
	class Allocator = std::allocator<Key>     // 指定分配器对象的类型

std::multiset 是含有 Key 类型对象有序集的容器。不同于 set ，它允许多个键拥有等价的值。用键比较函数 Compare 进行排序。搜索、插入和移除操作拥有对数复杂度。红黑树。

  • 随机访问——对数 O(\(log_2n\))
  • 插入或移除元素——对数 O(\(log_2n\))

1.7.3 使用场景

快速查找，允许重复值。

1.7.4 迭代器非法化

如果迭代器所指向的元素被删除，则该迭代器失效。其它任何增加、删除元素的操作都不会使迭代器失效。

1.8 std::map，一对一映射

1.8.1 概述

定义于头文件 <map>，类模板定义如下：

template<
    class Key,
    class T,
    class Compare = std::less<Key>,
    class Allocator = std::allocator<std::pair<const Key, T> >
> class map;

关联容器，底层基于红黑树，具有自动排序功能。特点是增加和删除结点对迭代器的影响很小，搜索、移除和插入操作拥有对数复杂度。

红黑树的存储空间比哈希表小，效率要查。

map 相较于 unordered_map 优势：

  1. map 保证遍历时 key 是有序的；
  2. map 可以做范围查找，而 unordered_map 不可以；
  3. map 的迭代器除非指向元素被删除，否则永远不会失效。unordered_map 的迭代器在对 unordered_map 修改时有时会失效；
  4. 因为3，所以对 map 的遍历可以和修改 map 在一定程度上并行（一定程度上的不一致通常可以接受），而对 unordered_map 的遍历必须防止修改；
  5. map 的迭代器可以双向遍历，这样可以很容易查找到当前 map 中刚好大于这个 key 的值，或者刚好小于这个 key 的值。

1.8.2 map 上的常见操作复杂度（效率）

  • 随机访问——对数 O(\(log_2n\))
  • 插入或移除元素——对数 O(\(log_2n\))

1.8.3 使用场景

一对多映射，基于关键字快速查找，不允许重复值。

1.8.4 迭代器非法化

如果迭代器所指向的元素被删除，则该迭代器失效。其它任何增加、删除元素的操作都不会使迭代器失效。

1.9 std::multimap，一对多映射

1.9.1 概述

定义于头文件 <map>，类模板定义如下：

template<
    class Key,							// 指定键的类型
    class T,							// 指定值的类型
    class Compare = std::less<Key>, 	// 指定排序规则
    class Allocator = std::allocator<std::pair<const Key, T> > // 指定分配器对象的类型
> class multimap;

multimap 是关联容器，底层基于红黑树，含有键值对的已排序列表，同时容许多个元素拥有同一键。按照应用到键的比较函数 Compare 排序。搜索、插入和移除操作拥有对数复杂度。

拥有等价键的键值对的顺序就是插入顺序，且不会更改。(C++11 起)

1.9.2 multimap 上的常见操作复杂度（效率）

  • 随机访问——对数 O(\(log_2n\))
  • 插入或移除元素——对数 O(\(log_2n\))

1.9.3 使用场景

一对多映射，基于关键字快速查找，允许重复值。

1.9.4 迭代器非法化

如果迭代器所指向的元素被删除，则该迭代器失效。其它任何增加、删除元素的操作都不会使迭代器失效。

1.9.5 示例

#include <iostream>
#include <map>

int main()
{
    // 创建并初始化 multimap 容器
    std::multimap<char, int> myMultimap{
        {'a', 10},
        {'b', 20},
        {'b', 15},
        {'c', 30}};

    // 输出 myMultimap 容器存储键值对的数量
    std::cout << myMultimap.size() << std::endl;

    // 输出 myMultimap 容器中存储键为 'b' 的键值对的数量
    std::cout << myMultimap.count('b') << std::endl;

    for (auto iter = myMultimap.begin(); iter != myMultimap.end(); ++iter)
    {
        std::cout << iter->first << " " << iter->second << std::endl;
    }

    return 0;
}

1.10 std::unordered_set，无序 set

1.10.1 概述

定义于头文件 <unodered_set>，类模板定义：

template<
    class Key, 								   //容器中存储元素的类型
    class Hash = std::hash<Key>,			 //确定元素存储位置所用的哈希函数
    class KeyEqual = std::equal_to<Key>,	//判断各个元素是否相等所用的函数
    class Allocator = std::allocator<Key>	//指定分配器对象的类型
> class unordered_set;

unordered_set 是含有 Key 类型唯一对象集合的关联容器。搜索、插入和移除拥有平均常数时间复杂度。

在内部，元素并不以任何特别顺序排序，而是组织进桶（本质就是链表）中。元素被放进哪个桶完全依赖其值的哈希。这允许对单独元素的快速访问，因为哈希一旦确定，就准确指代元素被放入的桶。

不可修改容器元素（即使通过非 const 迭代器），因为修改可能更改元素的哈希，并破坏容器。

1.10.2 unordered_set 上的常见操作复杂度（效率）

  • 随机访问——平均常数复杂度 O(\(n\))
  • 插入或移除元素——平均常数复杂度 O(\(n\))

如果采用合适的哈希函数，可能永远不会看到最坏情况。

1.10.3 使用场景

相比于 set 需要更快的访问。

1.11 std::unordered_multiset，无序 multiset

1.11.1 概述

定义于头文件 <unordered_set>，类模板定义：

template<
    class Key, 								   //容器中存储元素的类型
    class Hash = std::hash<Key>,			 //确定元素存储位置所用的哈希函数
    class KeyEqual = std::equal_to<Key>,	//判断各个元素是否相等所用的函数
    class Allocator = std::allocator<Key>	//指定分配器对象的类型
> class unordered_multiset;

unordered_multiset 是关联容器，含有可能非唯一 Key 类型对象的集合。搜索、插入和移除拥有平均常数时间复杂度。

元素在内部并不以任何顺序排序，只是被组织到桶（本质就是链表）中。元素被放入哪个桶完全依赖其值的哈希。这允许快速访问单独的元素，因为一旦计算哈希，它就指代放置该元素的准确的桶。

可以这样认为，unordered_multiset 除了能存储相同值的元素外，它和 unordered_set 容器完全相同。

1.11.2 unordered_multiset 上的常见操作复杂度（效率）

  • 随机访问——平均常数复杂度 O(\(n\))
  • 插入或移除元素——平均常数复杂度 O(\(n\))

1.11.3 使用场景

相比于 multiset 更快的访问。

1.12 std::unordered_map

1.12.1 概述

定于于头文件 <unordered_map>，类模板定义：

template<
    class Key,
    class T,
    class Hash = std::hash<Key>,
    class KeyEqual = std::equal_to<Key>,
    class Allocator = std::allocator< std::pair<const Key, T> >
> class unordered_map;

unodered_map 是关联容器，函数由带唯一键的键-值pair。搜索、插入和移除操作拥有平均常数复杂度。

元素在内部不以任何特定顺序排序，而是组织进桶中。元素放进哪个桶完全依赖其键的哈希。这允许对但单独元素的快速访问，因为一旦计算哈希，则它准确指代元素所放进的桶。

注意 unordered_map 重载了运算符 []，返回到映射到等于 key 的键的值的引用，若这种键不存在则进行插入。还可以更新既存值。

1.12.2 unordered_map 上的常见操作复杂度（效率）

  • 随机访问——平均常数复杂度 O(\(n\))
  • 插入或移除元素——平均常数复杂度 O(\(n\))

1.12.3 使用场景

相比 map 更快的访问。

1.12.4 示例

#include <iostream>
#include <string>
#include <unordered_map>

int main()
{
    // 创建三个 string 的 unordered_map, 映射到 string.
    std::unordered_map<std::string, std::string> u = {
        {"RED", "#FF0000"},
        {"GREEN", "#00FF00"},
        {"BLUE", "#0000FF"}};

    // 迭代并打印 unordered_map 的键和值.
    for (const auto &n : u)
    {
        std::cout << "Key: [" << n.first << "] Value: [" << n.second << "]\n";
    }

    // 添加新入口到 unordered_map.
    u["BLACK"] = "#000000";
    u["WHITE"] = "#FFFFFF";

    // 用键输出值.
    std::cout << "The HEX of color RED is: [" << u["RED"] << "]\n";
    std::cout << "The HEX of color BLACK is: [" << u["BLACK"] << "]\n";

    return 0;
}

2. 算法，Algorithm

string 和 vector 中的 size() 和 capacity()

size() 返回 vector 中的真实元素数量，而 capacity() 是 vector 的可容纳的元素数量，即预分配的内存空间。一个是真实，一个是最大上限。

另外，拥有 size() 和 capacity() 的模板类有 vector 和 string。

resize() 和 reserve() 的区别

resize()

其函数原型为

void resize(size_type count);									// (1)
void resize(size_type count, const value_type& value);		 // (2)

重设容器大小容纳 count 个元素。若当前大小大于 count，则减小容器为其首 count 个元素。若当前大小小于 count，1) 则后附额外的默认插入的元素；2）则后附额外的 value 的副本。

reserve()

其函数原型为

void reserve(size_type new_cap);

增加 vector 的容量到大于或等于 new_cap 的值。

  1. 若 new_cap 大于当前的 capacity()，则分配新存储空间，把之前 vector 中的对象通过复制构造函数复制过来，销毁之前的内存（包括所有迭代器，包含尾后迭代器和所有到元素的引用都被非法化）；
  2. 否则该方法不做任何事情。reverse() 不更改 vector 的 size，也没有迭代器或引用被非法化。

像这样理解：负责实际分配内存空间的 resize() 权限比 reserve() 高，resize() 的增加会同时修改 capacity()，而 capacity() 的增加仅修改 capacity()，不涉及内存，capacity() 的减少，当与 size() 冲突时，则什么也不做。

3. 迭代器，Iterator

3.1 概述

又称游标，用于提供一种方法顺序访问一个聚合对象中各个元素, 而又不需暴露该对象的内部表示。迭代器和指针非常类似，它是可以需要的任意类型，通过迭代器可以指向容器中的某个元素，如果需要，还可以对该元素进行读写操作。

由于迭代器于聚合对象的耦合，在一定程度上限制了它的广泛应用，一般仅用于底层聚合类。

常用的迭代器按照功能的强弱分为输入迭代器、输出迭代器、前向迭代器、双向迭代器、随机访问迭代器 5 种。

3.2 迭代器和指针的区别

迭代器不是指针，是类模板，表现得像指针。本质是封装了原生指针，是指针概念的一种提升，提供比指针更高级的行为，相当于一种智能指针，可以根据不同类型的数据结构来实现不同的自增自减操作。

迭代器返回的是对象的引用而不是对象的值，所以也需要通过 * 解引用才能获得指向对象的值。

3.3 迭代器产生的原因

迭代器的访问方式就是把不同集合的访问逻辑抽象出来，使得不用暴露内部的结构而达到循环遍历集合的效果。

3.4 如何正确使用迭代器删除元素（阅文 C++ 一面）

考察迭代器失效的问题。

  1. 对于顺序容器 vector、deque ，使用 erase() 删除元素后，删除位置后面每个元素的迭代器都会失效，同时这些元素会向前移动一个位置，erase() 会返回下一个有效的迭代器。正确的删除方法是

    if()
{
    it = vector.erase(it);
}
else
{
    ++it;
}

  2. 对于关联容器set、map ，使用 erase() 删除元素后，当前元素的迭代器失效，但由于其结构是红黑树，删除树中某一个结点，不会影响下一个元素的迭代器，所以只需在使用 erase() 递增当前迭代器即可。正确的删除方法是

    #include <iostream>
#include <map>
   
int main()
{
    std::map<int, std::string> c = {
        {1, "one"},
        {2, "two"},
        {3, "three"},
        {4, "four"},
        {5, "five"},
        {6, "six"}};
   
    // 从 c 擦除所有奇数
    for (auto it = c.begin(); it != c.end();)
    {
        if (it->first % 2 == 1)
        {
            c.erase(it++);
        }
        else
        {
            ++it;
        }
    }
    for (auto &p : c)
    {
        std::cout << p.second << ' ';
    }
   
    return 0;
}

  3. 对于 list，它使用了不连续的内存空间，并且它的 erase() 方法也会返回下一个有效的迭代器，因此，上面两种方法都可以使用。

容器适配器

迭代器适配器

所谓迭代器适配器，其本质也是一个模板类，比较特殊的是，该模板类是借助 5 种基础迭代器实现的。

本质上讲，迭代器适配器仍属于迭代器，可以理解为是基础迭代器的“翻新版”或者“升级版”。同时，“xxx 迭代器适配器”通常直接称为“xxx 迭代器”。

在 C++11 标准中，迭代器适配器有 4 类，如下图所示。

名称 功能
反向迭代器（reverse_iterator） 又称“逆向迭代器”，其内部重新定义了递增运算符（++）和递减运算符（--），专门用来实现对容器的逆序遍历。
安插型迭代器（inserter或者insert_iterator） 通常用于在容器的任何位置添加新的元素，需要注意的是，此类迭代器不能被运用到元素个数固定的容器（比如 array）上。
流迭代器（istream_iterator / ostream_iterator） 流缓冲区迭代器（istreambuf_iterator / ostreambuf_iterator） 输入流迭代器用于从文件或者键盘读取数据；相反，输出流迭代器用于将数据输出到文件或者屏幕上。 输入流缓冲区迭代器用于从输入缓冲区中逐个读取数据；输出流缓冲区迭代器用于将数据逐个写入输出流缓冲区。
移动迭代器（move_iterator） 此类型迭代器是 C++ 11 标准中新添加的，可以将某个范围的类对象移动到目标范围，而不需要通过拷贝去移动。
